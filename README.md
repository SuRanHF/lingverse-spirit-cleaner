# 神识清理 (LingVerse Spirit Cleaner)

适用于 `ling.muge.info` 的浏览器用户脚本，自动消耗神识、处理商人、雇佣护道、冥想恢复、铭文洗练、藏宝图刷图等，支持全自动闭环运行。

## 安装

1. 安装 Tampermonkey 或其他用户脚本管理器
2. 打开脚本安装地址：

```text
https://raw.githubusercontent.com/SuRanHF/lingverse-spirit-cleaner/main/lingverse-spirit-cleaner.user.js
```

3. 确认安装后进入 `https://ling.muge.info/game.html`，右下角出现控制面板即安装成功

## 功能

### 核心循环
- **自动清理神识**：设置保留神识、间隔、倍率，点一次"开始清理"循环执行
- **神识监测**：神识不足自动转入监测模式，等待自然恢复后自动继续
- **自动冥想**：神识不够时自动冥想恢复，支持高级冥想优先，冥想完成后自动回探索
- **道韵检查**：开始前检查道韵加成状态

### 战斗与护道
- **弱怪自战**：按战力对比自动判定，安全时无需护道直接战斗
- **自动护道**：支持最低价/合击/单独模式，可设灵石上限和重试次数
- **战后恢复**：战斗后自动检查血量，残血自动触发恢复

### 商人
- **传说才买** / **按条件购买** / **直接离去** 三种策略
- 支持品质关键词、价格上限、品质优先排序、严格/宽松匹配

### 自动恢复
- 回血/回灵可分别设置优先级顺序（灵力疗伤 → 丹药 → 宗门）
- 宗门快速治疗、灵气疗伤自动切换
- 装备耐久低于阈值自动一键维修

### 死亡处理
- 引渡复活全自动，复活后恢复 HP/MP 再继续
- 支持复活后跳转到指定地图区域

### 其他自动流程
- **虚空淬体**：自动维持 buff，用尽自动购买补充，连续失败自动关闭
- **隐秘符**：探索前自动使用，用尽自动购买，连续失败自动关闭
- **本命吞噬**：自动吞噬同类型装备或材料
- **自动收徒**：监听世界聊天自动收徒，处理徒弟问道/护道/历练请求
- **自动试炼塔**：自动挑战、选择最优 buff，结束后自动切回探索
- **自动藏宝图**：批量刷图、数量/间隔可配，用完后自动切回探索
- **铭文洗练**：十连洗练、按目标属性/品质/数值自动筛选，命中后自动切回探索
- **企业微信通知**：三通道 webhook 推送清理开始、商人购买、铭文命中、死亡复活等事件

### 面板
- 六标签页分类、可拖拽、可收起、可调节大小
- 设置自动保存到浏览器，重启不丢失
- 内置更新公告面板，定时检查云端新版本

## 配置说明

### 商人购买
- `商品关键词`：多个关键词用空格、逗号、顿号或分号分隔，支持品质名（普通/优良/稀有/史诗/传说）
- `品质优先`：先买高品质，同品质买便宜的
- `严格匹配`：品质 AND 名字同时满足才买；关闭后品质 OR 名字满足其一即买

### 自动恢复
- `恢复项目`：回血+回灵 / 只回血 / 只回灵 / 关闭
- `回血顺序` / `回灵顺序`：拖拽排序优先级
- `主动恢复`：开启后脚本主动调用 API 恢复，不依赖游戏内置恢复

### 铭文洗练
- 属性：攻击 / 防御 / 气血 / 神识
- 品质：不限 / 稀有及以上 / 史诗及以上 / 传说
- 命中模式：任一目标达成 / 全部目标达成

### 更新检测
脚本面板每 60 秒检查一次云端 `release.json`，发现新版本弹出更新公告。脚本管理器也会通过 `@updateURL` 自动更新。

## 本地文件说明

- `lingverse-spirit-cleaner.user.js` — 脚本本体
- `release.json` — 更新公告
- `versions/` — 历史版本备份

发布工具（`publish.ps1`、`一键发布.bat`）和服务器部署脚本分别位于本地和服务端仓库，不在此仓库中。
